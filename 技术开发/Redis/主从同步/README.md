# 全量同步流程

1. 主节点执行 BGSAVE，生成 RDB 快照文件
2. 主节点将 RDB 文件发送给从节点
3. 从节点清空旧数据，加载 RDB 文件
4. 主节点将 RDB 生成期间的新写命令 缓存在 复制缓冲区（replication buffer） 中
5. RDB 传输完成后，主节点将缓冲区中的写命令发送给从节点
6. 从节点执行这些命令，完成数据同步
7. 进入 命令传播阶段：主节点持续将新写命令异步发送给从节点

---

# 增量复制（部分重同步）流程

1. 主节点检查复制积压缓冲区（replication backlog）中是否包含从节点请求的 offset
2. 如果包含 → 主节点直接发送从 offset 开始的命令给从节点
3. 从节点接收并执行这些命令
4. 同步完成，继续 命令传播

### 关键组件填空提示（可用于流程图标注）

- PSYNC 命令：用于发起同步请求（Redis 2.8+）
- runid：主节点唯一标识
- offset：复制偏移量
- RDB 文件：全量同步的数据快照
- 复制积压缓冲区（replication backlog）：固定长度的环形缓冲区，用于支持增量复制
- 复制缓冲区（replication buffer）：为每个从节点单独维护，用于全量同步期间暂存新命令
- 命令传播：主节点将写命令持续发送给从节点
