# 分析
功能模型用数据流图来表示。
## 数据流图(Data Flow Diagram)

### 四要素

1. 数据流，箭头标明数据流向，标注数据流名称
2. 处理 P(输入和输出，数据的加工与转换)，圆角矩形或圆形
3. 数据存储 D(处理后处理或者处理前读取)，两条直线、缺右边的矩形
4. 外部实体 E(人、物、外部系统，数据提供者或消费者)：数据源或终点，矩形

### 建模步骤

1. 分析业务情况，绘制反应当前物理模型的数据流图
2. 推导出等价的逻辑模型 DFD
3. 设计新的逻辑系统，生成数据字典和基元描述
4. 建立人机接口，提出可供选择的目标系统物理模型 DFD
5. 确定各种方案的成本和风险，进行对比分析
6. 选定一种方案
