# 分析 用例模型
用例模型主要用于描述系统的**功能需求**，即系统从用户角度看来应该做什么。它由**用例图 (Use Case Diagram)** 和详细的**用例描述 (Use Case Specification)** 组成。

## 用例图 (Use Case Diagram)

**绘图元素说明：**

- **系统边界 (System Boundary)**：一个方框，代表待开发系统的范围。用例放在框内，参与者放在框外。
- **参与者 (Actor)**： stick figure，代表与系统交互的外部用户、角色或其他系统。
- **用例 (Use Case)**： 一个椭圆，代表系统提供的一个完整的功能单元。
- **关联 (Association)**： 参与者和用例之间的直线，表示两者有交互。
- **«include»**： 箭头实线，表示一个用例**必然包含**另一个用例的行为。例如，“下单”**必然包含**“支付”。
- **«extend»**： 箭头虚线，表示一个用例**可能在特定条件下扩展**另一个用例的行为。扩展用例是可选的吗。

**用例图示例：**

![](技术开发/软件工程/系统分析与设计%20面向对象/1.png)

**图解说明：**

1. **参与者**：我们识别出两个主要参与者——`**顾客**`和`**管理员**`。
2. **用例**：我们列出了系统为参与者提供的核心功能，如`**浏览图书**`、`**下单**`、`**管理图书**`等。
3. **关系**：

- `**下单**`用例**必须包含** `**支付**`用例（`**«include»**`）。
- `**评价图书**`用例**可能扩展** `**下单**`用例（`**«extend»**`），这意味着评价这个行为发生在下单之后，并且是可选的（不是每次下单后都必须评价）。

## 用例描述 (Use Case Specification)

用例图只是概要，每个用例都需要配以详细的文字描述，即**用例描述**。它通常以表格形式呈现。

**用例名称：** 下单

|   |   |
|---|---|
|**项目**|**描述**|
|**用例ID**|UC-5|
|**参与者**|顾客（主要）|
|**前置条件**|顾客已登录且购物车中有商品|
|**后置条件**|系统创建新订单，清空购物车中的对应商品|
|**主成功场景**|1. 顾客点击“去结算”。  <br>2. 系统显示订单确认页（包含商品、价格、收货地址）。  <br>3. 顾客确认订单并提交。  <br>4. 系统**包含**“支付”用例。  <br>5. 支付成功后，系统生成订单，保存到数据库，并清空购物车。  <br>6. 系统向顾客显示订单成功页面。|
|**扩展场景**|4a. 支付失败：  <br>1. 系统提示顾客支付失败原因。  <br>2. 返回订单确认页，顾客可重试或取消。|
|**特殊需求**|网络超时时间应小于30秒。|

# 分析 领域模型
领域模型描述系统业务范围内的**核心概念**、**概念的属性**以及**概念之间的关系**。它不关心软件实现，只关心业务本身。它通常表现为一个**类图**，但这里的类是“概念类”。

## 类图

**绘图元素说明：**

- **概念类 (Conceptual Class)**： 一个长方形，分为三层：类名、属性、方法。**在分析阶段，领域模型通常只写类名和关键属性，没有方法**。
- **关联 (Association)**： 类之间的连线，表示它们存在业务上的联系。线上可以标注关联的名称（如“拥有”）和多重性（Multiplicity）。
- **多重性 (Multiplicity)**： 表示一个类可以有多少个实例与另一个类的实例关联。

- `**1**` (有且只有一个)
- `**0..***` (0个或多个)
- `**1..***` (1个或多个)
- `*****` (多个)

类图示例：

![](技术开发/软件工程/系统分析与设计%20面向对象/2.png)

**图解说明：**

1. **概念类**：我们识别出业务中的核心概念，如`**Customer**`（顾客）、`**Book**`（图书）、`**Order**`（订单）等。
2. **属性**：为每个类添加了关键的业务属性。例如，`**Book**`有`**isbn**`, `**title**`, `**price**`等。
3. **关联和多重性**：

- 一个`**Customer**` **拥有** 一个`**ShoppingCart**` (`**1**` - `**1**`)。
- 一个`**ShoppingCart**` **包含** 多个`**CartItem**` (`**1**` - `*****`)。
- 一个`**CartItem**` **是** 一种`**Book**` (`**1**` - `**1**`)。
- 一个`**Customer**`可以**下达**多个`**Order**` (`**1**` - `*****`)。
- 一个`**Order**` **包含** 多个`**OrderLine**` (`**1**` - `*****`)，这是为了处理一个订单包含多种图书的情况。
- 一个`**Order**` **对应** 一个`**Payment**` (`**1**` - `**1**`)。
