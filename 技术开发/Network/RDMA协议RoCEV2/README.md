## 用户态驱动 libverbs

![](技术开发/Network/RDMA协议RoCEV2/1.png)

由于要 bypaas 内核，之前网卡的驱动是在内核态调用，现在需要在用户态调用。这个用户态驱动就是 libverbs 库。

### send&recv

在 Send & Receive 操作中，发送端和接收端的 APP 都需要参与数据接收过程。接收端需要告知 RDMA 网卡接收数据的目标地址，这样网卡收到数据时才知道要把数据拷贝到什么位置。

- server 端:

- 使用 `ibv_post_recv` 方法投递一个 recv 任务到网卡的队列里面去，这个任务描述了网卡在接收到数据之后，应该把数据写到那块内存上。
- 使用 `ibv_poll_cq` 方法，用轮询的方式从完成队列里获取事件，检查 recv 任务是否执行完成。执行完成就说明网卡已经把数据写到我们之前提供的内存上了。

- client 端:

- 使用 `ibv_post_send` 方法投递一个 send 任务到网卡的队列里面去，这个任务描述网卡需要发送的数据是什么。
- 使用 `ibv_poll_cq` 方法，用轮询的方式从完成队列里获取事件，检查 send 任务是否执行完成。执行完成就说明网卡已经把数据发给接收方了。

### read&write

是一种单端行为，就是说对端的 CPU 不需要参与通信过程，也不感知何时有数据写入、数据何时接收完毕。

### QP queue pair(SQ+RQ)

- 在RDMA技术中**通信的基本单元是QP，而不是节点。**
- 每个进程都可以使用若干个QP，而每个本地QP可以“关联”一个远端的QP。
- 每个节点的每个QP都有一个唯一的编号，称为QPN（Queue Pair Number），通过QPN可以唯一确定一个节点上的QP。
